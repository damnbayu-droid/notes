# Technical Orientation (v15.0.3-PRODUCTION-STABLE)

This project is in a **Neural-Perfect** state, optimized for **Structural DOM Integrity**, **Zero-Block Performance**, and **Production-Grade Ergonomics**.

### 🧬 Architectural Mandates
1. **Structural Discovery Protocol (v7.6.2)**: All shared intelligence MUST be delivered in a semantically valid DOM tree. Text-only nodes MUST be wrapped in `<p>` blocks or relevant schema tags to ensure 100% parsing success for non-visual AI agents.
2. **Privacy-First Sitemap**: The `sitemap.xml` MUST EXCLUSIVELY index nodes where `is_discoverable = true`. Regular `is_shared` links MUST NOT be broadcast to public crawlers.
3. **Fast Open SSR Architecture**: All root routing shells MUST implement instantaneous SSR layout mapping via `DashboardSkeleton.tsx` for zero-latency initial load visualization.
4. **Data Integrity Mandate (Protocol BUFF)**: All intelligence editing interfaces MUST implement ultra-redundant persistence:
    - **Force-Save on Exit**: Immediate, non-debounced sync during `onClose` and `beforeunload`.
    - **Black-Box Recovery**: Secondary local-storage buffer to survive cloud sync failures.
5. **Neural Auth Bus (Singleton Protocol)**: All user data MUST route through the unified `initializeAuth` memory pool within `hooks/useAuth.ts` to strictly prohibit `AbortError: Lock broken` race conditions between unmounted nodes and `gotrue-js`.
6. **Neural Media Protocol (v7.7.0)**: All intelligence nodes MUST prioritize hosted assets over embedded data, with absolute WebP/AVIF enforcement.
7. **Admin & Sidebar Alignment Protocol (v7.9.0)**: In shrunken mode (80px), all icons MUST be absolutely centered. All administrative components MUST adhere to the **10px baseline typography** and collapsible sidebar interaction models to prevent viewport overflow.
8. **Performance Guard (Zero-Block Protocol)**: External prompts (e.g., Support Modal, Storage Limits) MUST be deferred. Safe Sync protocols MUST only ADD local intelligence without wiping remote architectures (Append-Only).
9. **Shared Backend**: Connected to Supabase project `dfxhfutflhnxjjpbqscj`. Data integrity and RLS policies are strictly enforced.
